# CPQ 配置器需求梳理

## 1. 目前配置器里有哪些内容

### 1.1 区域认证
- CE
- ANSI
- 其他区域认证（按市场扩展）

### 1.2 基本参数
- 属性：不可选、可展示
- 特点：随配置选择结果动态变化
- 示例：整机尺寸、重量、关键性能参数

### 1.3 配置选择（研发设计相关）
- 发动机
- 辅具
- 其他设计相关选项

### 1.4 制造属性（工艺生产相关）
- 液压油
- 发动机油
- 齿轮油
- 防冻液
- 润滑脂
- 其他制造工艺属性

## 2. 客户需求中需要进入配置器的内容

### 2.1 装运方式
建议纳入配置器，并作为可选业务项维护。

可选案例：
- 整机发运
- 整机 + 拆轮胎发运
- 货箱后门单独发运
- 底盘 + 货箱单独发运

### 2.2 随车材料
建议纳入配置器，并支持按语种进行选择。

可选案例：
- 产品手册（多语种）
- 其他随车文件（可扩展）

## 3. 其他类型是否需要进入选配器

### 3.1 服务条款
建议：
- 默认不进入核心技术选配流程
- 可作为商务配置模块，或在报价/订单阶段选择

### 3.2 付款条款
建议：
- 默认不进入核心技术选配流程
- 放在销售订单或合同环节管理

### 3.3 其他条款
建议按以下原则判断是否进入选配器：
- 若会影响产品设计、制造、BOM 或交付形态：进入配置器
- 若仅影响商务交易或合同条款：不进入配置器，放在报价/合同模块

## 4. 建议的分类边界（落地参考）

- 技术配置层（进入配置器）：区域认证、配置选择、制造属性、影响交付形态的装运方式
- 交付配置层（可选进入配置器）：随车材料（多语种）
- 商务配置层（不进入核心选配）：服务条款、付款条款及其他合同类条件

## 5. 后续可执行动作

- 将“装运方式”和“随车材料”定义为标准配置项，补充数据字典与默认值
- 在配置器界面增加“交付与资料”分组
- 对商务条款建立独立模块，与配置结果做关联而非混放
