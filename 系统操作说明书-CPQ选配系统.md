# CPQ选配系统操作说明书

## 1. 文档目的
本说明书用于指导业务人员、售前人员、产品管理人员和测试人员正确使用当前 CPQ（Configure, Price, Quote）系统，完成从底表维护到产品选配、价格核算、配置留档与导出的全流程操作。

## 2. 系统定位（业务视角）
当前系统是一套前端本地化 CPQ 工作台，支持以下核心业务目标：

1. 维护选配底表（工程机型、销售机型、价格表）。
2. 按三阶段完成客户配置（产品线 -> 机型 -> 配置选项）。
3. 实时计算报价并识别“待确认价格”的定制项。
4. 保存未完成/已完成选配记录，支持后续续配。
5. 导出配置数据（JSON/CSV）用于下游流程（如 PLM/ETO 对接准备）。

## 3. 适用角色与职责

### 3.1 工程/产品管理员
1. 维护工程机型基础数据。
2. 发布销售机型并控制可见配置项。
3. 建立和维护价格表。

### 3.2 销售/售前
1. 按客户需求完成产品选配。
2. 查看实时价格汇总。
3. 保存选配记录，必要时发起定制（ETO）流程。

### 3.3 测试/运维支持
1. 执行数据导入导出。
2. 恢复默认底表。
3. 验证不同用户数据隔离效果。

## 4. 菜单与功能总览
系统左侧菜单包含 6 个功能区：

1. 测试工具：数据备份、恢复、重置。
2. 工程机型：查看工程机型并发布销售机型。
3. 价格表：创建/编辑/复制价格表，维护选项价格。
4. 销售机型：管理对外销售机型，控制配置可见性与顺序。
5. 选配历史：查看、续配、删除、导出历史配置。
6. 产品选配：执行正式选配流程与报价。

## 5. 标准业务流程（推荐）
建议按以下顺序使用系统：

1. 在“工程机型”确认基础机型数据完整。
2. 在“价格表”建立每个工程机型对应的价格表。
3. 在“销售机型”发布并修订对外机型（名称、可见项、默认价格表）。
4. 在“产品选配”按三阶段完成客户配置并保存。
5. 在“选配历史”查看结果，按需导出 JSON/CSV 或继续编辑未完成配置。

## 6. 各模块操作说明

### 6.1 测试工具

#### 6.1.1 导出数据
适用场景：环境迁移、备份归档、测试快照留存。

操作步骤：
1. 进入“测试工具”。
2. 点击“导出数据”。
3. 系统下载当前用户数据备份文件（JSON）。

说明：
1. 导出包含底表与历史配置数据。
2. 文件名带用户ID与日期，便于追踪。

#### 6.1.2 导入数据
适用场景：恢复备份、复制测试环境。

操作步骤：
1. 点击“导入数据”。
2. 选择备份 JSON 文件。
3. 导入成功后系统自动刷新并重载状态。

注意事项：
1. 导入文件格式错误会提示失败。
2. 建议先导出现有数据再执行导入。

#### 6.1.3 重置默认
适用场景：清除导入底表，恢复初始数据。

操作步骤：
1. 点击“重置默认”。
2. 二次确认后执行。
3. 系统清除底表本地数据并刷新，从默认 JSON 重新加载。

说明：
1. 该操作主要针对底表恢复。
2. 建议操作前先做导出备份。

### 6.2 工程机型

#### 6.2.1 查看工程机型详情
1. 进入“工程机型”。
2. 在目标机型行点击“查看”。
3. 在弹窗中查看配置组、类别、默认值等明细。

业务意义：
1. 工程机型是销售机型与价格表的源头。
2. 先确认工程机型准确，再进行发布和报价维护。

#### 6.2.2 发布销售机型
1. 在目标工程机型行点击“发布销售机型”。
2. 系统自动创建对应销售机型并跳转“销售机型”进入编辑。

业务意义：
1. 销售机型继承工程机型配置结构。
2. 可面向市场做展示与可选范围的二次治理。

### 6.3 价格表

#### 6.3.1 新建价格表
1. 进入“价格表”。
2. 点击“新建价格表”。
3. 选择关联工程机型。
4. 系统自动生成该机型配置项价格清单。
5. 填写价格表名称、描述、币种并维护单项价格。
6. 点击“保存”。

规则说明：
1. 价格表按工程机型关联。
2. 销售机型在选配时仅可选择同工程机型的价格表。

#### 6.3.2 编辑价格表
1. 在列表点击“编辑”。
2. 修改表头信息或逐项调整价格。
3. 保存后立即生效。

#### 6.3.3 复制价格表
1. 在列表点击“复制”。
2. 基于副本做差异化调整并保存。

适用场景：
1. 多地区币种价格。
2. 不同时间窗口价格策略。

#### 6.3.4 删除价格表
1. 点击删除按钮。
2. 若价格表被销售机型引用，系统会阻止删除。

### 6.4 销售机型

#### 6.4.1 查看与编辑销售机型
1. 进入“销售机型”。
2. 点击“修订”进入编辑弹窗。
3. 可执行：
   1. 修改销售机型名称。
   2. 绑定默认价格表。
   3. 控制配置组、类别、选项的显示/隐藏。
   4. 调整类别/选项顺序（按钮或拖拽）。
4. 点击“保存”。

业务意义：
1. 决定客户选配界面可见内容。
2. 用于屏蔽不对外销售选项，提升选配效率和准确性。

#### 6.4.2 删除销售机型
1. 在列表点击“删除”。
2. 二次确认后删除。

注意事项：
1. 删除不可撤销。
2. 建议仅在确认不再使用时执行。

### 6.5 产品选配（核心业务流程）

产品选配为三阶段流程，支持中途保存。

#### 阶段一：选择产品线
1. 进入“产品选配”。
2. 通过层级导航（类似文件夹）定位到目标产品线叶子节点。
3. 点击目标产品线进入下一阶段。
4. 如需暂存可点击“保存当前层级”，系统将保存进度并可在历史中续配。

#### 阶段二：选择机型
1. 系统展示与当前产品线关联的可选机型。
2. 可能出现“工程机型(销售机型)”并列显示。
3. 选择目标机型进入配置阶段。
4. 可点击“保存当前进度”记录阶段状态。

#### 阶段三：配置选项
1. 按配置组逐项选择。
2. 只读组（如基础参数）不可手动修改。
3. 可计价组选项会实时影响右侧价格汇总。
4. 支持“自定义”输入特殊需求（价格显示为待确认）。
5. 点击“保存选配”完成留档。
6. 若存在自定义且配置完整，按钮会提示“保存并发起ETO流程”。

### 6.6 选配历史

#### 6.6.1 查看历史记录
1. 进入“选配历史”。
2. 查看保存时间、产品线、机型、状态、配置号、总价等。

#### 6.6.2 继续选配
1. 对未完成记录点击“继续选配”。
2. 系统按记录阶段恢复现场：
   1. 产品线确认中 -> 返回产品线阶段。
   2. 机型确认中 -> 返回机型阶段。
   3. 配置确认中 -> 返回选项阶段。

#### 6.6.3 查看详情与导出
1. 点击“详情”查看完整配置结构和价格拆分。
2. 在详情弹窗可导出：
   1. JSON：适合系统对接/结构化存档。
   2. CSV：适合表格分析与线下流转。

#### 6.6.4 删除记录
1. 点击删除图标并确认。
2. 记录删除后不可恢复。

## 7. 价格与配置规则（业务口径）

### 7.1 价格构成
总价 = 基础价格 + 已选可计价项加价 + 定制待确认项（如有）

表现形式：
1. 无自定义项：显示确定总价。
2. 有自定义项：显示“确定总价 + ?”。

### 7.2 配置完整性
系统会检查所有可见配置类别是否都有“标准选项”或“自定义输入”。

结果影响：
1. 完整配置可归档为“选配完成”。
2. 未完成则记录为“配置确认中”。

### 7.3 配置号规则
系统可生成配置号用于追踪：
1. 标准配置：`std` 前缀。
2. 含自定义配置：`tmp` 前缀。

## 8. 状态定义（历史记录）

1. 产品线确认中：仅确认到产品线层级。
2. 机型确认中：产品线已确认，机型尚未最终确定。
3. 配置确认中：机型已选，配置尚未完整闭环。
4. 选配完成：配置完整并已保存。

## 9. 数据存储与安全边界

### 9.1 存储方式
1. 当前版本主要使用浏览器本地存储（localStorage）。
2. 首次或重置后会从系统内置 JSON 初始化。

### 9.2 用户隔离
1. 系统为每个浏览器用户分配 userId。
2. 数据按 userId 前缀隔离，避免本地互相覆盖。

### 9.3 操作建议
1. 重要数据定期导出备份。
2. 大批量导入前先保留旧快照。
3. 同机多人演示时关注当前 userId 是否符合预期。

## 10. 常见问题与处理建议

### 10.1 为什么看不到某些配置项？
可能原因：
1. 该配置项在销售机型管理中被隐藏。
2. 当前阶段尚未进入对应配置组。

建议：
1. 由管理员在“销售机型”检查显隐设置。

### 10.2 为什么价格表不能删除？
原因：
1. 该价格表仍被某销售机型引用。

建议：
1. 先在销售机型中改绑价格表，再删除。

### 10.3 为什么总价显示 `+ ?`？
原因：
1. 存在自定义配置项，价格待人工确认。

建议：
1. 作为 ETO/工程评审输入，后续补齐报价。

### 10.4 导入后数据未生效怎么办？
建议：
1. 确认导入文件结构为有效 JSON。
2. 观察导入提示，必要时重新导入。
3. 先执行导出备份，再重置并重导。

## 11. 日常操作清单（SOP）

### 11.1 每日开工检查
1. 确认当前 userId。
2. 确认底表是否为最新版本。
3. 确认关键价格表是否可用。

### 11.2 选配交付前检查
1. 核对产品线与机型是否正确。
2. 核对价格表币种与场景是否一致。
3. 检查是否存在自定义项（若有需标注待确认）。
4. 保存并在历史中复核一次。
5. 按需导出 JSON/CSV 归档。

### 11.3 变更管理建议
1. 调整销售机型显隐前先备份。
2. 价格表大改建议先复制再修改。
3. 发布前由业务与产品双人复核。

## 12. 版本边界说明
本说明书面向当前系统实现，重点能力为本地化数据管理与前端选配闭环。
若后续接入后端主数据、权限体系、审批流或正式 ETO 工单流，应补充对应章节并升级操作规范。

## 13. 约束规则（PLM -> CPQ）

### 13.1 规则数据结构（建议标准）
`rules` 挂在机型对象下（工程机型/销售机型均可携带），支持两类规则：`enable`、`exclude`。

```json
{
   "rules": [
      {
         "id": "ER_001",
         "type": "enable",
         "name": "柴油动力限定电器版本",
         "description": "选择柴油动力后，仅允许指定电器版本",
         "when": [
            { "category_code": "POWER", "option_code": "POWER_DIESEL" }
         ],
         "then": [
            {
               "category_code": "ELEC",
               "allowed_option_codes": ["ELEC_24V", "ELEC_48V"]
            }
         ],
         "priority": 10,
         "enabled": true
      },
      {
         "id": "XR_001",
         "type": "exclude",
         "name": "高空平台与窄底盘互斥",
         "description": "该组合在工程上不可同时成立",
         "items": [
            { "category_code": "PLATFORM", "option_code": "PLATFORM_HIGH" },
            { "category_code": "CHASSIS", "option_code": "CHASSIS_NARROW" }
         ],
         "priority": 20,
         "enabled": true
      }
   ]
}
```

字段说明：
1. `id`：规则唯一标识，建议由 PLM 统一维护。
2. `type`：`enable`（缩减可选范围）或 `exclude`（禁止组合）。
3. `priority`：冲突时按优先级从小到大执行。
4. `enabled`：是否启用，便于灰度发布。
5. `enable.when`：触发条件（全部满足才触发）。
6. `enable.then`：触发后可选范围（白名单）。
7. `exclude.items`：不可同时成立的组合项。

### 13.2 系统行为（已实现）
1. 导入：系统在加载机型时自动规范化 `rules` 结构并过滤非法规则。
2. 实时计算：每次选项变化都会重算可选范围、冲突集合和修复建议。
3. 约束传播：启用规则触发后，相关特征值范围自动收敛并联动到界面。
4. 冲突修复：检测到排除冲突时，系统自动尝试最小修复并给出建议。
5. 前端防错：禁选项不可点击，悬浮可见“不可选原因”，顶部显示规则提示与冲突信息。
